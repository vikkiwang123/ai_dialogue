# 📖 AI对话监控助手 - 使用指南

## 🎯 关于API的问题

**重要：您不需要任何大模型的API！**

这个插件的工作原理是：
- ✅ **本地监控**：直接在浏览器中监控网页上的对话内容
- ✅ **本地存储**：所有数据存储在您的浏览器本地
- ✅ **无需API**：不需要OpenAI、Claude等任何API密钥
- ✅ **完全免费**：插件本身完全免费使用

插件通过分析网页的DOM结构来提取对话内容，就像您手动复制粘贴一样，只是自动化了。

## 🚀 如何开始使用

### 第一步：确认插件已启用

1. 点击浏览器工具栏中的插件图标
2. 查看状态显示是否为"监控中"
3. 如果显示"已暂停"，请到设置页面开启

### 第二步：访问AI平台

访问以下任一支持的平台：
- **ChatGPT**: https://chat.openai.com （推荐，最稳定）
- **Claude**: https://claude.ai
- **Copilot**: https://copilot.microsoft.com
- **Gemini**: https://gemini.google.com

### 第三步：开始对话

1. 在AI平台上正常开始对话
2. 发送消息后，AI回复时插件会自动记录
3. **重要**：需要等待AI完全回复完成后再发送下一条消息

### 第四步：查看记录

1. 点击插件图标查看今日统计
2. 点击"生成今日总结"查看学习收获
3. 右键点击插件图标 → 选项，查看详细数据

## 🔍 为什么Gemini没有记录？

可能的原因：

1. **页面结构变化**：Gemini的页面结构可能与我们适配器中的选择器不匹配
2. **消息未完全加载**：AI回复还在生成中
3. **插件未正确注入**：需要刷新页面

### 调试步骤

1. **打开开发者工具**（F12）
2. **查看控制台**，应该能看到：
   ```
   AI对话监控助手 - Content Script 已加载
   检测到平台: gemini
   开始监控平台: gemini
   ```
3. **发送一条消息**，观察控制台是否有：
   ```
   消息已保存: user/assistant [消息内容前50字]
   ```
4. **如果没有看到这些日志**，说明：
   - 插件可能未正确加载（刷新页面）
   - 或者Gemini的页面结构已变化（需要更新适配器）

## 📝 使用技巧

### 最佳实践

1. **推荐使用ChatGPT**：ChatGPT的适配最稳定，建议先用它测试
2. **等待回复完成**：确保AI完全回复后再发送下一条
3. **定期查看统计**：每天结束时查看一下今日总结
4. **导出备份**：定期导出数据备份，防止丢失

### 常见问题

**Q: 为什么有些消息没记录？**
A: 可能是消息太短（少于10个字符）或者页面结构变化

**Q: 数据存储在哪里？**
A: 存储在浏览器本地，可以通过设置页面查看和导出

**Q: 可以同时监控多个平台吗？**
A: 可以！打开多个标签页，每个平台都会独立监控

**Q: 插件会影响页面性能吗？**
A: 影响很小，使用了防抖和节流优化

# 🔧 插件激活指南

## 关于"服务工作进程 (不活动)"状态

**这是正常现象！** 浏览器扩展的Service Worker只有在需要时才会激活。

### 如何激活服务

1. **访问支持的AI平台**
   - 打开 https://chat.openai.com 或 https://gemini.google.com
   - 插件会自动激活

2. **点击插件图标**
   - 点击浏览器工具栏中的插件图标
   - 打开popup界面也会激活服务

3. **手动激活（如果需要）**
   - 在扩展管理页面，点击"服务工作进程"旁边的"检查视图"
   - 或者点击"重新加载"按钮

## ✅ 验证插件是否正常工作

### 方法1：检查控制台日志

1. 访问 ChatGPT：https://chat.openai.com
2. 按 F12 打开开发者工具
3. 切换到 Console 标签
4. 应该能看到：
   ```
   AI对话监控助手 - Content Script 已加载
   检测到平台: chatgpt
   开始监控平台: chatgpt
   ```

### 方法2：测试消息记录

1. 在ChatGPT中发送一条消息
2. 等待AI回复
3. 控制台应该显示：
   ```
   消息已保存: user [您的消息]
   消息已保存: assistant [AI的回复]
   ```

### 方法3：查看popup统计

1. 点击插件图标
2. 应该能看到今日统计（如果已有数据）
3. 如果没有数据，显示为0是正常的

## 🐛 如果服务无法激活

### 问题1：点击"检查视图"没有反应

**解决方案：**
- 刷新扩展管理页面
- 或者直接访问AI平台，服务会自动激活

### 问题2：访问AI平台后仍显示"不活动"

**解决方案：**
1. 确认插件已启用（开关是开启状态）
2. 刷新AI平台页面
3. 检查是否有错误提示

### 问题3：控制台没有日志

**解决方案：**
1. 确认当前页面是支持的AI平台
2. 检查manifest.json中的host_permissions是否包含当前域名
3. 尝试重新加载插件：
   - 在扩展管理页面点击"重新加载"
   - 然后刷新AI平台页面

## 📝 正常使用流程

1. ✅ **安装插件**（已完成）
2. ✅ **访问AI平台**（如ChatGPT）

- 建议先用ChatGPT测试，适配最稳定
